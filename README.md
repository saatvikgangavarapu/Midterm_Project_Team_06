## Project Overview:
This midterm university model project is a Java 19 application built using Netbeans and Swing GUI. The objective is to build a well-connected application that simulates a university application, serving mutliple entities that exist in a typical university. Our project implements the administrators, the faculty, and the students. By implementing a role-based access, we were able to separate the functionalities all through the same application. This allows the different users to interact with the gui elements that apply only to them, rather than having to navigate through panels that they don't use in their day-to-day operations. 

## Key Features:
Key features of this project include the ability to manage students, faculty, courses, and enrollments within a Digital University System. The system supports secure login functionality for multiple user accounts, each assigned specific roles and permissions to ensure proper authentication and role-based access control

### Faculty Usecase:
Faculty users can manage courses, including viewing specific courses and updating the course details. They also manage both their own and their student’s profiles, including interests and academic progress. Additionally, they can generate performance reports containing students’ class grades.  

### Administrator Usecase:
Administrators can log in to their account and access the Admin Dashboard, which allows them to create, delete and view accounts from the Administration Page. Additionally, on the Registration page they can register accounts of students, admin and faculty and  manage, view, update and delete student and faculty accounts through the Manage Student and Manage Faculty pages respectively, and finally view their own Profile page.

### Student Usecase:
The Student role allows users to manage coursework (enroll and drop classes), access academic records, tell them their eligibility for graduation, review transcripts, and track accumulated credits towards a completed degree.
